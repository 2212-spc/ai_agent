# 🤖 AI Agent - 智能对话助手

一个功能强大的 AI Agent 平台，集成了 RAG（检索增强生成）、MCP 工具调用、上网搜索、长期记忆系统等能力，提供类似 ChatGPT 和 Coze 的专业级用户界面。

![](https://img.shields.io/badge/FastAPI-0.110+-green)
![](https://img.shields.io/badge/DeepSeek-API-orange)
![](https://img.shields.io/badge/LangGraph-Agent-purple)

## 🌟 核心特性

- 🧠 **智能长期记忆** - 自动提取、去重、向量化检索，让 AI 真正记住用户
- 🤖🤖🤖 **多智能体系统**（最新） - 多个专家智能体协作，任务自动分解，结果智能汇总，支持Prompt模板管理
- 📚 **会话管理** - 完整的对话历史检索、继续对话、记忆共享控制
- 🔍 **向量化检索** - ChromaDB 支持，语义相似度搜索，带置信度评估
- 🤖 **LangGraph Agent** - 多步骤规划、智能路由、工具调用
- 📄 **RAG 知识库** - 文档上传、向量检索、智能分块
- 🌐 **上网能力** - 网页搜索、内容提取、天气查询，智能工具调用
- 🎯 **Prompt模板管理**（最新） - 数据库存储、动态加载、占位符替换，灵活定制智能体行为

- ✅ **记忆共享控制** - 可选择是否跨会话共享长期记忆

### 🎯 Prompt模板管理系统（最新）
- ✅ **数据库存储** - Prompt模板存储在数据库中，支持动态管理
- ✅ **智能体关联** - 每个模板关联到特定智能体（analysis_specialist、summarization_specialist等）
- ✅ **激活机制** - 每个智能体只有一个激活模板，切换模板自动更新
- ✅ **占位符替换** - 支持 `{user_query}`、`{task_description}`、`{analysis_context}` 等占位符
- ✅ **默认模板** - 系统提供默认模板作为参考，可创建自定义模板
- ✅ **模板管理** - 支持创建、更新、激活、删除模板（默认模板受保护）
- ✅ **动态加载** - 运行时从数据库加载激活模板，无需重启服务
- ✅ **向后兼容** - 如果数据库中没有激活模板，自动使用硬编码的默认prompt

**支持的智能体：**
- `analysis_specialist` - 分析专家
- `summarization_specialist` - 总结专家
- `verification_specialist` - 验证专家
- `retrieval_specialist` - 检索专家（主要用于说明，不直接使用LLM）

**占位符说明：**
- `{user_query}` - 用户查询
- `{task_description}` - 任务描述
- `{analysis_context}` - 分析上下文
- `{full_context}` - 完整上下文
- `{final_answer}` - 最终答案
- 支持自定义占位符，通过 `get_agent_prompt` 函数的 `**kwargs` 传递

## 🚀 快速开始

### 前置要求
- Python 3.10 或更高版本
- pip 或 conda 包管理器
- DeepSeek API Key（[获取地址](https://platform.deepseek.com/)）

### 1. 克隆项目

```bash
git clone <your-repo-url>
cd ai_agent
```

### 2. 创建虚拟环境（推荐）

**使用 Conda:**
```bash
conda create -n ai-agent python=3.10 -y
conda activate ai-agent
```

**或使用 venv:**
```bash
python -m venv venv
# Windows:
venv\Scripts\activate
# Linux/Mac:
source venv/bin/activate
```

### 3. 安装依赖

```bash
cd backend
pip install -r requirements.txt
```

> ⏱️ **注意**：首次安装会下载嵌入模型（sentence-transformers），大约需要 5-10 分钟，请耐心等待。

### 4. 配置环境变量

**Windows (PowerShell):**
```powershell
$env:DEEPSEEK_API_KEY = "sk-your-api-key-here"
```

**Linux/Mac:**
```bash
export DEEPSEEK_API_KEY="sk-your-api-key-here"
```

> 💡 **提示**：如需永久保存，可以创建 `.env` 文件或添加到系统环境变量。

### 5. 启动后端服务

```bash
cd backend
python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

或使用 PowerShell 脚本（Windows）:
```powershell
cd backend
.\重启增强版服务器.ps1
```

看到以下输出表示启动成功：
```
INFO:     Uvicorn running on http://0.0.0.0:8000 (Press CTRL+C to quit)
INFO:     Started reloader process
INFO:     Started server process
INFO:     Waiting for application startup.
INFO:     Application startup complete.
```

### 6. 打开前端界面

在浏览器中打开：
- **登录页面**：`frontend/login.html`
- **注册页面**：`frontend/register.html`（或从登录页点击注册链接）
- **聊天界面**：`frontend/chat.html`（登录后自动跳转）
- **LangGraph Agent 界面**：`frontend/agent_chat.html`（推荐，登录后自动跳转）
- **对话历史页面**：`frontend/conversation_history.html`（从聊天界面点击 📚 按钮访问）
- **对话设置页面**：`frontend/conversation_settings.html`（从聊天界面点击 ⚙️ 按钮访问）

或者使用 Live Server（VS Code 插件）打开 `frontend/login.html`

### 使用多智能体系统（最新功能）

多智能体系统适合处理复杂的研究任务，通过多个专家智能体的协作，提供更全面、更准确的答案。

**使用场景：**
- 📚 **研究助手**：需要检索、分析、总结的复杂问题
- 🔍 **深度调研**：需要多角度分析的主题研究
- 📊 **数据分析**：需要提取、分析、可视化的数据任务

**使用方法：**
```
# 调用多智能体 API 端点
POST /chat/multi-agent/stream
{
  "messages": [{"role": "user", "content": "请帮我研究一下扩散模型的最新进展"}],
  "use_knowledge_base": true,
  "use_tools": true,
  "execution_mode": "sequential"  // 或 "parallel"
}
```

**工作流程：**
1. **协调器规划** - 分析任务，分解为子任务，选择合适的智能体
2. **检索专家** - 从知识库检索（带置信度评估，低置信度自动跳过）、智能工具调用（LLM自动判断需要哪些工具）
3. **分析专家** - 深度分析检索到的内容，提取关键信息（支持自定义Prompt模板）
4. **总结专家** - 智能选择信息源，整合所有信息，生成结构化报告（支持自定义Prompt模板）
5. **验证专家** - 质量检查（可选，支持自定义Prompt模板）

**新增特性：**
- 🎯 **智能工具调用** - LLM自动分析用户问题，判断需要调用哪些工具（天气、搜索等）
- 📊 **置信度评估** - 知识库检索结果带置信度评估，低相关性内容自动跳过
- 🔄 **信息源智能选择** - 总结专家会根据信息质量自动选择最相关的信息源
- 📝 **Prompt模板管理** - 支持为各智能体自定义Prompt模板，动态加载无需重启

**优势：**
- 🎯 **任务分解更精准** - AI 自动分解复杂任务
- ⚡ **执行更高效** - 专业智能体并行处理，智能工具调用减少无效操作
- 🔍 **结果更全面** - 多角度协作，信息更完整
- ✅ **质量更可靠** - 多智能体验证，答案更准确
- 🎨 **高度可定制** - Prompt模板管理系统，灵活调整智能体行为

### 使用长期记忆系统

**自动记忆提取：**
系统会在每次对话后自动提取重要信息，例如：
- 用户提到姓名、职业等事实信息 → 存储为 `fact` 类型
- 用户表达偏好（喜欢、不喜欢） → 存储为 `preference` 类型
- 用户提到事件、计划 → 存储为 `event` 类型

**记忆自动去重：**
系统会智能检测相似记忆并合并，例如：
```
第一次对话："我叫张三"
第二次对话："我的名字是张三"
→ 自动合并，避免重复存储
```

**记忆检索：**
AI 会在回答问题时自动检索相关记忆：
```
用户："你还记得我的名字吗？"
AI：根据记忆检索 → "当然记得，您叫张三"
```

3. 使用搜索框搜索特定对话内容

**继续之前的对话：**
1. 在历史记录中点击要继续的对话
2. 系统自动加载历史消息
3. 可以无缝继续之前的对话

**删除会话：**
1. 在历史记录页面点击 **🗑️** 按钮
2. 确认删除后，该会话的所有消息将被永久删除

**配置记忆共享：**
1. 点击聊天界面右上角的 **⚙️ 会话设置** 按钮
2. 切换 **跨会话共享记忆** 开关：
   - **开启**：AI 可以访问所有会话中存储的长期记忆
   - **关闭**：AI 只能访问当前会话的记忆（会话隔离）
3. 设置会自动保存并生效

### 使用Prompt模板管理系统

**查看和管理模板：**
1. 访问 Prompt 管理页面（通常在设置或管理界面中）
2. 查看所有智能体的模板列表
3. 每个智能体可以有多個模板，但只有一个处于激活状态

**创建自定义模板：**
1. 选择要创建模板的智能体（如 `analysis_specialist`）
2. 填写模板名称、描述和内容
3. 在模板内容中使用占位符，如 `{user_query}`、`{task_description}` 等
4. 保存后模板会自动激活（如果这是该智能体的第一个模板）

**激活模板：**
1. 在模板列表中找到要激活的模板
2. 点击"激活"按钮
3. 系统会自动将该智能体的其他模板设为非激活状态
4. 激活后立即生效，无需重启服务

**编辑模板：**
1. 点击模板的"编辑"按钮
2. 修改模板内容、名称或描述
3. 保存后立即生效

**删除模板：**
- 只能删除非默认模板
- 默认模板（`is_default=true`）受保护，无法删除
- 删除后如果该智能体没有其他激活模板，系统会使用硬编码的默认prompt

**占位符使用示例：**
```
你是一个资深的分析专家。请分析以下内容：

用户问题：{user_query}
任务要求：{task_description}
待分析内容：{analysis_context}

请提供详细的分析报告。
```

**注意事项：**
- 模板内容支持 Markdown 格式
- 占位符使用单花括号 `{variable_name}` 格式
- 如果占位符未定义，系统会记录警告但不会中断执行
- 系统会自动处理双花括号转义（`{{` → `{`）

## 🐛 常见问题

### Q: 首次安装很慢？
A: 正常现象，需要下载嵌入模型（约 500MB）和依赖包，请耐心等待。

### Q: 数学公式不显示？
A: 确保网络畅通，KaTeX 库需要从 CDN 加载。可以按 F12 查看控制台错误。

### Q: 搜索功能不可用？
A: 检查网络连接，确保可以访问 DuckDuckGo。部分地区可能需要代理。

### Q: DeepSeek API 调用失败？
A: 检查：
1. API Key 是否正确设置
2. 是否有剩余额度
3. 网络是否畅通

### Q: 上传文档失败？
A: 确保：
1. 文件格式为 PDF
2. 文件大小合理（< 100MB）
3. 文件未加密

### Q: 记忆没有被记住？
A: 检查：
1. 确保对话中包含明确的事实或偏好信息
2. 记忆提取需要一定重要性，过于琐碎的信息可能不会被提取
3. 查看会话设置，确认记忆共享开关是否开启

### Q: 如何在会话间隔离记忆？
A: 在会话设置中关闭"跨会话共享记忆"开关，AI 将只能访问当前会话的记忆。

### Q: 如何删除对话历史？
A: 访问对话历史页面，点击要删除的会话右侧的 🗑️ 按钮即可删除。

### Q: 注册时密码太长导致错误？
A: 系统已优化，支持任意长度密码。如果密码超过72字节，系统会自动进行安全处理（SHA-256哈希+Base64编码），确保密码安全存储。

### Q: 登录页面一直刷新？
A: 这通常是因为token验证失败导致的循环。解决方案：
1. 清除浏览器缓存和localStorage
2. 重新登录
3. 如果勾选了"记住我"，确保token有效

### Q: 忘记密码怎么办？
A: 当前版本为演示模式，暂不支持密码重置功能。如需重置，可以：
1. 删除数据库中的用户记录
2. 重新注册账号

### Q: 可以同时登录多个账号吗？
A: 当前版本使用localStorage存储认证信息，同一浏览器只能登录一个账号。如需切换账号，请先退出登录。
